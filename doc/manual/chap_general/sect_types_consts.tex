\section{Data types and constants}

The SCDC library provides and uses definitions for the following data types and constants:
\reflst{%
\reflstdtyp{scdcint_t}{Predefined data type used for (almost) all specifications of integer values (\eg, parameters, return values, \dots) throughout the entire library.}
\reflstcnst{SCDC_SUCCESS}{Constant of data type \dtyp{scdcint_t} signaling the success of an operation.}
\reflstcnst{SCDC_FAILURE}{Constant of data type \dtyp{scdcint_t} signaling the failure of an operation.}
\reflstdtyp{scdc_nodeport_t}{Data type for storing a node port handle (see \ref{sect:server_nodeport}).}
\reflstcnst{SCDC_NODEPORT_NULL}{Null constant for node port handles.}
\reflstdtyp{scdc_dataprov_t}{Data type for storing a data provider handle (see \ref{sect:server_dataprov}).}
\reflstcnst{SCDC_DATAPROV_NULL}{Null constant for data provider handles.}
\reflstdtyp{scdc_dataset_t}{Data type for storing a dataset handle (see \ref{sect:client_dataset}).}
\reflstcnst{SCDC_DATASET_NULL}{Null constant for data provider handles.}
}
